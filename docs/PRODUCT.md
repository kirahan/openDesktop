# OpenDesktop 产品功能总览（规格定义）

> **定位**：本机 **Electron 等桌面应用的受管调试控制面**——用 **Core** 持久化 App/Profile、拉起带调试端口的子进程、提供 **Control API** 与 **会话级 CDP 网关**；可选 **浏览器内 Web UI（Studio）** 作为同源客户端。

> **规格来源**（仓库根目录 `openspec/specs/`）：`opendesktop-core`、`opendesktop-control-api`、`opendesktop-cdp-gateway`、`opendesktop-studio`。

---

## 1. Core 运行时与数据模型（`opendesktop-core`）


| 能力              | 说明                                                                                                                                             |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **守护进程**        | 长驻 **Core**；可仅通过 CLI 在无图形界面下启动/停止；默认监听 **本机回环**（如 `127.0.0.1`）。                                                                                |
| **就绪与健康**       | 提供可判断「Core 已就绪」的方式（如 HTTP 健康检查）；就绪前不应把依赖 Core 的会话创建标成成功。                                                                                       |
| **优雅停止**        | 停止时尽量优雅；超时后可强制；文档需说明对子进程策略。                                                                                                                    |
| **App 注册**      | 可执行路径、工作目录、环境/参数模板等；持久化 JSON，含 `**schemaVersion`**，**原子写**。                                                                                    |
| **Profile**     | 绑定已有 App，可覆盖/追加环境变量与启动参数；引用不存在 App 时拒绝创建。                                                                                                      |
| **Session 状态机** | 会话状态至少覆盖：`pending` / `starting` / `running` / `killed` / `failed`（或语义等价集合）；`**killed`** 表示用户停止或子进程退出/断线，会话记录仍保留在列表直至 Core 重启；新会话 **始终新 UUID**。 |
| **子进程与调试**      | 按 App/Profile 拉起目标进程；支持为调试 **注入远程调试相关参数**（具体策略由设计固定）；**捕获 stdout/stderr** 并与会话关联（含时间戳等可观测信息）。                                                  |
| **CLI ↔ API**   | 创建/列出/停止会话、列出 App/Profile、Core 状态等 **核心操作** CLI 与 API **语义等价**；**不依赖 Studio** 也能完成全流程。                                                         |
| **CLI（App 优先）** | 支持 **`od <appId> list-window|metrics|snapshot`**（`topology` 为兼容别名）；默认按应用解析最新活跃会话；`--session`、`--format`、`od doctor` 与约定退出码。 |

## 2. Control API 与安全（`opendesktop-control-api`）


| 能力           | 说明                                                                                            |
| ------------ | --------------------------------------------------------------------------------------------- |
| **HTTP API** | 版本化前缀（如 `**/v1`**）：列表/创建会话、查询详情、停止会话、列出 App 与 Profile、健康检查等；**JSON + UTF-8**。                 |
| **鉴权**       | 除 **health/version** 等公开端点外，**Bearer（或设计固定方式）+ 本机 token**；token 来自环境变量或文件；**日志中不打印完整 token**。 |
| **流式日志/状态**  | 至少 **WebSocket 或 SSE** 之一，用于订阅会话日志或状态变更；只实现一种时，设计里说明另一种的预留。                                   |
| **审计**       | 对会话创建/停止/失败、App/Profile 变更等记 **审计事件**（时间、操作类型、主体、资源 ID）；首版可为本地追加文件；**可导出或查看**。                |
| **版本信息**     | 提供 `**/v1/version`（或等价）**，返回 Core 与 API 版本；行为与是否免鉴权在文档中明确。                                    |


---

## 3. CDP 网关（`opendesktop-cdp-gateway`）


| 能力                | 说明                                                                            |
| ----------------- | ----------------------------------------------------------------------------- |
| **逻辑 CDP 入口**     | 每个活跃调试会话有 **稳定逻辑 URL**（路径或查询携带 **session id**），客户端 **不必**只依赖手写子进程真实端口。        |
| **与 DevTools 兼容** | CDP 就绪后，通过逻辑端点可完成 CDP 所需 HTTP 握手（例如访问 **Chrome 兼容的 JSON 列表**等），得到合法响应，而非连接被拒。 |
| **端口分配**          | 为本机远程调试分配 **未占用端口**；分配失败则会话 **failed** 且错误可解析。                                |
| **网络边界**          | 默认 **仅本机回环**；若允许绑定外网须默认关闭、需显式配置并文档警示。                                         |
| **会话结束**          | Session 停止/失败后，逻辑端点 **失效**；旧 URL 不应再完成 CDP 握手。                                |

## 4. Web UI / Studio（`opendesktop-studio`）


| 能力          | 说明                                                                                           |
| ----------- | -------------------------------------------------------------------------------------------- |
| **托管与同源**   | SPA 由 **Core 同一 HTTP 服务** 提供，与 `**/v1` API 同源**；终端用户默认 **不依赖**单独前端 dev server。               |
| **客户端角色**   | UI **不**作为会话编排唯一真相源；**只**通过同源 Control API（或文档允许的 base URL）与 Core 通信；可选 **远程 Core** 基址并提示风险。  |
| **静态资源与路由** | 生产构建产物由 Core 托管；**SPA fallback**（非 `/v1`、非静态文件时回 `index.html`），深链刷新不 404。                    |
| **会话列表与详情** | 列表至少含：Session ID、状态、关联 Profile/App 名（若 API 提供）、创建时间；可从列表进 **详情**，查看 **逻辑 CDP URL** 或 **复制**。 |
| **错误态**     | Core 不可达时 **明确错误**，与「空列表」区分。                                                                 |
| **日志视图**    | 订阅流式日志（或设计允许的轮询兜底）；可 **按级别/关键词** 过滤（首版可极简）；与 CLI 日志 **同一管道语义**（原始文本一致）。                      |
| **技术栈**     | **React + TypeScript**，构建优先 **Vite**；首版 **不**用 Electron/Tauri 作壳。                            |
| **分发**      | Web UI **随 Core 一起发布**；不要求用户单独安装「Studio 桌面应用」；README 说明 **启动 Core → 浏览器打开 → 可选 CLI `open`**。 |

## 5. 能力关系（示意）

```
                    ┌─────────────────────────────────────┐
                    │              Core                    │
                    │  App / Profile / Session / 子进程   │
                    │  Control API (/v1) + token          │
                    │  CDP Gateway → 子进程真实端口        │
                    └──────────┬──────────────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         ▼                     ▼                     ▼
   ┌───────────┐        ┌─────────────┐       ┌──────────────┐
   │ CLI       │        │ 浏览器 SPA   │       │ Agent/脚本    │
   │ 无 GUI 完成 │        │ Studio      │       │ CDP/自动化    │
   │ 全流程     │        │ 同源 API     │       │ 逻辑 CDP URL  │
   └───────────┘        └─────────────┘       └──────────────┘
```

## 6. 规格与实现

本文档描述的是 **OpenSpec 主规格中的 SHALL 要求**。具体实现是否覆盖每一条，需按代码与 UI **对照**；若需维护「规格 vs 实现」差距表，可在 OpenSpec 中单独开 change 跟踪。

### 6.1 已实现扩展（工程交付，非主规格全文）

- **窗口列表 / 指标 / 日志导出**：`GET /v1/sessions/:id/list-window|metrics`（`/topology` 为兼容别名），`GET .../logs/export`。
- **Agent 面**：`GET /v1/agent/sessions/:id/snapshot`，`POST .../actions`（含截图、eval 等，受 Profile `allowScriptExecution` 与限流约束）。
- **版本能力位**：`GET /v1/version` 含 `capabilities`。
- **配置**：`OPENDESKTOP_AGENT_API`、`OPENDESKTOP_EXTENDED_LOGS`、`OPENDESKTOP_AGENT_RPM`。
